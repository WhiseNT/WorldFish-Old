# WorldFish LLM 世界观创建指南

本指南用于指导 LLM 直接创建与 WorldFish 当前数据框架兼容的世界观内容。目标不是写一篇散文式设定，而是产出一个可以被前端世界观构建页、时间线、地图、设定管理与后续推演流程直接消费的结构化世界观。

## 1. 目标

LLM 生成的世界观必须同时满足四个条件：

1. 世界观本身完整，有清晰的时代背景、核心矛盾和关键实体。
2. 时间结构完整，至少包含锚定时间、纪元划分，以及关键政治实体或团体的存续信息。
3. 设定结构可拆分，能够映射到角色、物品、组织、地理、能力、其他设定六类。
4. 输出可直接映射到 WorldFish 的字段，而不是依赖人工二次整理。

## 2. 必须对齐的数据框架

LLM 必须围绕以下六个区域组织内容：

1. world_info
2. entities
3. events
4. settings.items
5. settings.mapData
6. settings.calendars

### 2.1 world_info

必须提供：

- name：世界观名称
- description：世界观的一段总述，说明世界运行逻辑与核心冲突
- era：时代背景，例如“群星殖民时代晚期”“诸侯割据后的蒸汽神权时代”
- anchor_time：一个最适合作为时间轴锚点的时间描述，例如“新曙历 431 年”或“帝国崩溃后第 17 年”

要求：

- era 不是空泛标签，必须能概括政治、技术、社会氛围。
- anchor_time 必须能和事件、纪元、政治实体的时间表述互相对齐。

### 2.2 entities

实体至少应覆盖：

- 关键人物
- 主要国家、政权、组织、宗教、联盟、商业集团
- 重要地点、城市、区域、遗迹
- 关键物品、技术、制度、超凡体系载体

每个实体必须包含：

- name
- type
- attributes

其中，政治实体或团体必须优先保留以下时间属性中的一种或多种：

- 成立时间
- 建立时间
- 建国时间
- 灭亡时间
- 解散时间
- 存续时间
- 在位时间
- 统治时间

示例：

```json
{
  "name": "北境联邦",
  "type": "国家",
  "attributes": {
    "政体": "联邦共和制",
    "成立时间": "曙光纪元 87 年",
    "存续时间": "87 ~ 312",
    "核心领土": "寒湾、白岭、镜湖平原"
  }
}
```

### 2.3 events

事件必须是可以挂到时间轴上的关键节点，而不是琐碎背景句子。

每个事件必须包含：

- name
- description
- date
- entities

要求：

- date 必须使用与世界主时间体系兼容的表述。
- entities 必须引用该事件直接相关的实体名称。
- 至少包含建国、战争、制度改革、灾变、技术突破、神话断裂、宗教分裂等能改变世界状态的事件。

### 2.4 settings.items

设定条目必须拆成可管理的卡片，而不是把所有设定堆进 description。

category 只能取以下之一：

- character
- item
- organization
- geography
- ability
- other

每条设定建议同时提供：

- name
- category
- description
- aliases
- detailContent

要求：

- description 用于列表摘要。
- detailContent 用于详情页，应比 description 更完整。
- 不要把地图关系和历法内容塞进 settings.items，它们有独立区域。

### 2.5 settings.mapData

必须拆分为：

- regionRelations：区域、边疆、地理板块的关系
- countryRelations：国家、阵营、政权之间的关系
- importantLocations：关键城市、关隘、遗迹、战略节点

要求：

- 用事实描述，不要写成散文。
- 重要地点最好可以被事件和实体引用。

### 2.6 settings.calendars

这里不是装饰字段，而是时间线的骨架。

每条历法必须包含：

- name
- type：只能是“纪元”或“纪年”
- baseTime
- timeRange
- unit
- ratio
- calendarType
- description

约束：

- 纪元用于大的时代分段，例如王朝更替、文明阶段、灾后重建期。
- 纪年用于更细的政权或统治期记录。
- timeRange 必须可比较，例如“120 ~ 240”“-300 ~ -120”“431 ~ 无”。
- 如果一个世界观存在多个时间体系，必须说明彼此换算关系。

示例：

```json
{
  "name": "曙光纪元",
  "type": "纪元",
  "baseTime": "0",
  "timeRange": "0 ~ 312",
  "unit": "年",
  "ratio": "×1",
  "calendarType": "未开启",
  "description": "以第一座恒光塔点燃作为纪元起点。"
}
```

## 3. 时间结构硬性要求

任何符合本框架的世界观，至少要让时间线能展示以下三层：

1. 历法体系摘要：有哪些历法，如何计时，彼此如何换算。
2. 纪元划分：大的时代区间如何分段，分段依据是什么。
3. 政治实体或团体存续：国家、政权、联盟、组织在什么时间开始、结束或延续至今。

如果缺少这三层，世界观就无法在当前 UI 中形成有意义的时间结构。

## 4. 写作原则

### 4.1 不要使用占位符

禁止出现：

- 某某帝国
- 某组织
- 某时代
- 若干年后
- 古老传说中曾有

所有关键对象必须命名。

### 4.2 不要只写背景氛围

“这是一个战争频发的黑暗时代”这种句子不够。必须补齐：

- 谁在战争
- 战争开始于何时
- 哪些政体被改变
- 哪些地点重要
- 哪些事件推动局势转折

### 4.3 不要让时间表达失配

如果 anchor_time 写“新曙历 431 年”，而事件和政体时间都写“第三次王朝更替后不久”，就无法对齐。至少要让关键事件和政体能落到统一的时间区间里。

### 4.4 让设定彼此引用

好的世界观不是条目堆砌，而是：

- 事件影响政治实体
- 政治实体控制地理区域
- 地理区域塑造资源与技术
- 技术或超凡体系反过来改变战争与政体

## 5. 推荐生成流程

建议 LLM 按以下顺序生成：

1. 先确定世界核心冲突与时代背景。
2. 定义主时间体系与锚定时间。
3. 划分 2 到 5 个纪元或时代阶段。
4. 定义主要政治实体，并为其补成立、统治或灭亡时间。
5. 补关键人物、地点、组织、技术、规则。
6. 生成关键事件，并把事件挂回已有实体。
7. 最后整理为 world_info、entities、events、settings 三大块。

## 6. 可直接复用的 System Prompt

```text
你是 WorldFish 的世界观架构师。你的任务不是写散文，而是创建一个可直接映射到 WorldFish 世界观框架的结构化世界观。

你必须同时生成：
1. 世界观基本信息（名称、总述、时代背景、锚定时间）
2. 关键实体（人物、国家、组织、地点、物品、体系）
3. 关键事件（必须能落到统一时间轴）
4. 设定条目（按 character、item、organization、geography、ability、other 分类）
5. 地图关系（区域关系、国家关系、重要地点）
6. 历法与纪元体系（必须能支撑时间线）

硬性要求：
- 不得输出占位符名称
- 不得只写氛围，不写结构
- 国家、政权、组织、王朝、联盟等政治实体必须尽量保留成立、灭亡、存续或在位时间
- 事件时间、锚定时间、历法区间必须互相兼容
- 纪元和纪年必须分清
- 所有内容都必须能被后续世界观推演使用
```

## 7. 可直接复用的 User Prompt 模板

```text
请为我创建一个符合 WorldFish 框架的原创世界观。

题材方向：{题材}
整体气质：{气质}
核心冲突：{核心冲突}
技术或超凡水平：{技术或超凡水平}
政治主题：{政治主题}
希望重点展开的对象：{重点对象}

请输出时务必确保：
1. world_info、entities、events、settings.items、settings.mapData、settings.calendars 六部分都完整
2. settings.items 使用 character、item、organization、geography、ability、other 分类
3. 至少给出一个 anchor_time
4. 至少给出 2 个纪元或时间阶段
5. 至少给出 2 个带明确存续时间的政治实体或团体
6. 所有关键事件都能挂接到时间线和实体
```

## 8. 生成后自检清单

在输出前，LLM 应逐项自检：

1. 世界观名称是否具体而非占位？
2. 是否存在明确时代背景和锚定时间？
3. 是否存在可比较的纪元或纪年区间？
4. 是否至少有两个政治实体或团体带显式起止时间？
5. 事件是否真的改变世界，而不是普通注释？
6. 设定条目是否已正确分类？
7. 地图关系是否与实体和事件互相引用？
8. 是否还有无法映射到当前 UI 的散落信息？

如果任一项答案是否，就继续补全后再输出。
